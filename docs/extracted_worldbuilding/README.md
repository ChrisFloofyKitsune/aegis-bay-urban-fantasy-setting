# Extracted Worldbuilding Documents

These files were extracted and distilled from the raw discussion logs for **Project Aegis Bay**.

## Source Logs

- `discussion_logs\openrouter_chat__world_prompt_discussion_dump.md`
- `discussion_logs\jenova_ai_chat__world_building_discussion_dump.jsonl`
- `discussion_logs\jenova_ai_chat__world_building_discussion_readable.md` *(generated readable, deduplicated transcript)*

## Status Guide

- **Current / Prompt-Aligned** — matches or directly supports the current `world_prompt.md`
- **Creator-Only** — backend logic intended for GM notes, not for the live prompt
- **Exploratory / Superseded** — valuable idea bank, but not current canon unless re-promoted later

## Files

### 1. `aegis_bay__current_canon_and_prompt_inputs.md`
**Status:** Current / Prompt-Aligned

Distills the late-stage OpenRouter discussion into a usable overview of the setting as it was being turned into the production world prompt: core premise, factions, neighborhoods, PHCs, DHA logic, opening structure, and anti-leakage aesthetic rules.

### 2. `aegis_bay__gm_screen_hidden_architecture.md`
**Status:** Creator-Only

Collects the hidden lore that was repeatedly marked as "behind the GM screen": global fracture history, the true origin of the Supernatural/Mythic split, the 3-axis ontology, and the DHA's institutional blind spot.

### 3. `aegis_bay__layer_1_5_exploration_brief.md`
**Status:** Current-adjacent / Exploration Tool

Preserves the Jenova-derived "Layer 1.5" exploration brief: a lighter-weight setting guide meant to help a player, writer, or AI understand the city without loading the full metaphysical backend.

### 4. `aegis_bay__exploratory_branches_and_superseded_models.md`
**Status:** Exploratory / Superseded

Preserves early high-value brainstorming from the Jenova discussion: seed-crystal veil spread, historical fracture candidates, alt-history branches, the older Integration x Ontological Stress grid, and the narrative-digestion thesis.

## Practical Note

The **OpenRouter log** is where most of the prompt-ready, Aegis-Bay-specific canon crystallized.

The **Jenova log** is where several deeper theoretical and exploratory frameworks were stress-tested. Some of those ideas were later simplified, hidden, or replaced in the production prompt. They are still worth keeping as a design archive.
